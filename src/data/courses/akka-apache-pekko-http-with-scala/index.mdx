<Fragment slot="goal">
### REST Easy

Distributed systems are a fact of life now. However, [distributed monoliths](https://www.microservices.com/talks/dont-build-a-distributed-monolith/) can grind your system and your dev speed to a halt. Although they're not perfect, microservices can solve a lot of your problems. Akka HTTP can help you spin up any HTTP or REST service, big or small, in minutes. It's [Akka Streams](https://rockthejvm.com/course/akka-streams)-based, fault-tolerant, [reactive](https://www.reactivemanifesto.org/), and powers the brilliant [Play Framework](https://www.reactivemanifesto.org/) and many of the services we take for granted.

This course will teach you how to write HTTP and REST APIs the right way with this powerful toolkit.

</Fragment>
<Fragment slot="question">
### After this course, you'll be able to:
- create any HTTP service of any complexity
- write REST APIs that can send and receive anything
- handle JSON with both high-control and hands-off, when you choose
- use 50+ directives and combinations for arbitrary server logic
- support JWT authorization in your projects
- secure your server with HTTPS
- test routes and server logic
- use WebSockets for back-and-forth communication
- support arbitrary file uploads to your server
- use 3 different client APIs to interact with any other REST service

</Fragment>
<Fragment slot="skills">
- Process any HTTP requests and extract headers and payloads
- Build HTTP microservices in low-level, high-control mode
- Marshal and unmarshal JSON in HTTP services
- Build HTTP microservices with high-level Directives
- Handle URL paths and extract query parameters
- Create routing logic and compose routes of any complexity
- Upload files of any size and create WebSockets
- Incorporate JWT authorization
- Handle HTTP rejections and recover from internal and remote exceptions
- Build HTTP clients in connection-level, host-level and request-level modes
- Stream HTTP request and response bodies
